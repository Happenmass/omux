Cliclaw 架构深度分析（Chat-Driven MainAgent）

> **架构更新 (2026-03-04)**: 当前架构为 Chat-Driven 模式。关键特征：
> - **已删除**: Planner, TaskGraph, Scheduler — 不再有多阶段规划/调度
> - **入口**: 用户通过 Web Chat UI 发送消息 → `MainAgent.handleMessage(content)`
> - **状态机**: IDLE ↔ EXECUTING 双状态，LLM 自主决定执行与终止
> - **服务层**: Express HTTP + WebSocket 服务器，端口 3120
> - **持久化**: SQLite 对话持久化 + 服务器重启恢复
> - **SignalRouter**: `stop()`, `resume()`, `isStopRequested()` 执行控制
> - **MainAgent 事件**: `state_change`, `log`
> - **工具**: 14 个内置工具 + 技能动态注入
> - **ContextManager 模块**: `compressed_history`, `memory`, `agent_capabilities`（已移除 `goal`, `task_graph_summary`）
>
> 以下部分图表为重构前的历史架构参考，已明确标注。

这是一个 元级 Agent 编排系统 —— 用一个 LLM（MainAgent）通过 tool use 循环来监控和控制另一个 Agent（Claude Code），通过 tmux 实现非侵入式交互。用户通过 Web Chat UI 与系统对话，MainAgent 以聊天模式自主决策。

┌────────────────┬───────────────────────────────────────────────────────────┐
│    特征         │                           描述                           │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 架构模式       │ Chat-Driven 元编排器 (Meta-Orchestrator)                  │
│                │ 用 LLM streaming tool use 控制 LLM Agent                  │
│                │ MainAgent 为中枢决策节点，IDLE ↔ EXECUTING 状态机         │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 用户交互       │ Web Chat UI → WebSocket → Express 服务器                 │
│                │ 流式文本输出 (assistant_delta) + 工具活动广播              │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 通信方式       │ 通过 tmux 桥接，非侵入式（Agent 不知道自己被控制）       │
│                │ AgentAdapter 接口抽象，可替换不同 Agent 实现               │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 决策方式       │ LLM streaming tool use 循环：stream → text delta 广播    │
│                │ → tool calls → 执行 → 继续，直到终止性工具或纯文本       │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 信号路由       │ SignalRouter: stop()/resume()/isStopRequested()          │
│                │ StateDetector 回调 → 信号分流: 快速通道 + MainAgent 通道  │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 状态检测       │ 三层：正则快速匹配 → LLM 语义分析 → 深度错误诊断        │
│                │ 输出仅 status/confidence/detail，不含 suggestedAction     │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 上下文管理     │ 模块化系统提示 + 持续对话历史 + SQLite 持久化             │
│                │ 两层保护: 记忆刷新 (60%) + LLM 压缩 (70%)                │
│                │ 服务器重启可从 SQLite 恢复对话状态                         │
├────────────────┼───────────────────────────────────────────────────────────┤
│ 错误恢复       │ MainAgent 自主决策：mark_failed /                         │
│                │ escalate_to_human，基于完整上下文推理                     │
├────────────────┼───────────────────────────────────────────────────────────┤
